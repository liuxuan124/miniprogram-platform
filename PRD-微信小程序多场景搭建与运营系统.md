## 2. 产品概述

本产品是一套面向品牌、商家、活动运营方的微信小程序搭建与运营管理系统。平台通过后台可视化配置能力，帮助运营人员完成小程序页面搭建、内容发布、会员运营、商品交易、订单处理、营销活动、活动报名、预约服务、素材管理和 AI Agent 配置，同时在小程序端向用户提供内容浏览、会员权益、商城购买、活动报名、预约服务、智能客服等服务。

产品核心价值是降低小程序运营和迭代成本，让非技术运营人员可以通过后台完成页面装修、业务配置和内容更新，并让商家在一个后台中完成从获客、转化、交易、履约到复购的闭环管理。

## 3. 背景与问题

中小品牌和服务型商家通常需要同时运营内容、活动、会员、商品、预约和客服，但传统小程序开发依赖研发排期，页面改版和活动上线周期长，运营数据和交易数据分散，AI 客服与业务系统也缺少统一配置入口。

因此需要建设一套低代码化、模块化、可运营的多场景小程序后台，使运营人员可以快速发布页面和活动，商务人员可以管理商品、订单和营销，管理员可以配置小程序基础能力、支付、物流、权限和 AI Agent。

## 4. 产品目标

1. 支持运营人员在后台完成小程序页面创建、装修、预览、保存草稿、版本管理和发布上线。
2. 支持内容、表单、会员、用户、商品、订单、营销、活动、预约等核心业务模块的日常运营。
3. 支持小程序端用户完成浏览内容、查看会员权益、购买商品、提交表单、预约服务、参与活动和咨询 AI 助手。
4. 支持微信小程序基础配置、微信支付、物流配送、订阅消息、底部导航、插件开关和角色权限管理。
5. 支持 AI Agent 从模型接入、Prompt 配置、知识库、沙盒测试、发布管理到运营监控的完整生命周期。

本产品统一按“完整增强版”作为目标范围，不再按不同产品层级拆分。验收口径以业务闭环完整、页面搭建能力强、营销能力丰富、小程序展示效果丰富、AI Agent 可对话并能推荐商品和文章为准。

增强版目标范围包括：

1. 完整基础电商能力：商品、购物车、订单、微信支付、发货、核销、退款、物流、售后和订单统计。
2. 完整内容发布能力：文章、图文、视频、分类、封面、推荐、上下架、富文本、小程序内容详情。

6. AI Agent 推荐闭环：用户对话、商品推荐、文章推荐、活动引导、会员权益解释、知识库问答、转人工兜底和运营监控。

## 8. 核心业务流程

### 8.1 页面发布流程

1. 运营人员登录后台。
2. 进入页面管理。
3. 新建页面或选择已有页面。
4. 进入页面装修器。
5. 从组件库添加 Banner、导航、商品、文章、活动入口、会员卡等组件。
6. 在属性面板配置标题、数据源、按钮、样式、跳转。
7. 保存草稿。
8. 预览小程序效果。
9. 发布上线。
10. 小程序端实时展示新版本。

### 8.2 商品交易流程

1. 商务运营在商品管理中创建商品。
2. 按商品类型填写实物、数字或服务类字段。
3. 商品上架后在小程序商城展示。
4. 用户进入商品详情并下单。
5. 用户确认订单并支付。
6. 系统接收微信支付回调。
7. 实物商品进入待发货，数字商品进入待核销，服务商品进入待确认。
8. 运营人员完成发货、核销或服务确认。
9. 用户可查看订单状态。
10. 如发生售后，进入退款审核流程。

### 8.3 活动报名流程

1. 运营人员创建活动，配置名称、时间、地点、名额、报名表单。
2. 活动入口可配置到首页或专题页。
3. 用户在小程序端查看活动详情。
4. 用户提交报名表单。
5. 后台生成表单提交记录。
6. 运营人员审核报名。
7. 审核通过后触发订阅消息。
8. 活动现场可进行核销。

### 8.4 预约服务流程

1. 管理员配置服务项目、日期、时间段和容量。
2. 用户在小程序端选择服务项目。
3. 用户选择日期和时间段。
4. 用户确认预约。
5. 后台生成预约记录。
6. 服务人员确认预约。
7. 预约前触发提醒通知。
8. 服务完成后可记录服务状态。

### 8.5 AI Agent 发布流程

1. 管理员选择模型并配置 API Key。
2. 编写 System Prompt、欢迎语、无法回答策略和行为策略。
3. 上传知识库文件并完成向量化。
4. 在沙盒中测试典型问题。
5. 查看测试评估，补充知识库或调整 Prompt。
6. 创建待发布版本。
7. 选择全量、灰度或定时发布。
8. 发布后在小程序 AI 助手中生效。
9. 在运营监控中查看对话量、满意度、意图分布和转化行为。

## 12. 非功能需求

### 12.1 性能

1. 后台页面首屏加载时间不超过 3 秒。
2. 列表查询接口响应时间 P95 不超过 800ms。
3. 页面发布后小程序端配置生效时间不超过 30 秒。
4. AI Agent 普通问答响应首 token 时间不超过 3 秒，完整回复不超过 10 秒。

### 12.2 可用性

1. 后台关键操作需有明确成功、失败或待处理反馈。
2. 删除、发布、退款、切换生产支付环境等高风险操作必须二次确认。
3. 表单、商品、支付、物流等配置需进行必填校验和格式校验。
4. 手机预览应与小程序真实展示尽量一致。

### 12.3 安全

1. AppSecret、API Key、支付密钥、证书等敏感信息需加密存储和脱敏展示。
2. 后台接口需鉴权，关键操作需记录操作日志。
3. 角色权限需按最小权限原则控制。
4. 支付回调需校验签名并防重放。
5. AI Agent 不应泄露系统 Prompt、API Key、用户隐私和后台敏感数据。

### 12.4 兼容性

1. 后台需兼容最新版 Chrome、Edge、Safari。
2. 小程序端需兼容主流 iOS 与 Android 微信版本。
3. 管理后台建议优先适配 1366px 及以上桌面分辨率。

## 14. 验收标准

1. 用户可以通过登录页进入后台，并完成主要导航切换。
2. 工作台能够展示指标、趋势、待办和快捷入口。
3. 页面管理可以新建页面、进入装修器、保存草稿、预览并发布。
4. 页面装修器能够完成组件选择、结构展示、属性配置和手机预览。
5. 内容、表单、会员、用户、商品、订单、营销、活动、预约、素材等模块均具备列表和核心操作入口。
6. 商品交易流程至少覆盖商品详情、确认订单、支付成功、后台发货/核销/退款审核。
7. 系统设置至少覆盖小程序、支付、物流、订阅消息、导航、插件和权限。
8. AI Agent 至少覆盖模型接入、Prompt、知识库、沙盒测试、发布和监控。
9. 小程序端至少覆盖首页、内容、会员、商城、AI 助手、我的五类核心页面。
10. 高风险操作具备确认或明确提示，敏感信息脱敏展示。

## 16. 风险与待确认问题

| 风险/问题 | 说明 | 建议 |
| --- | --- | --- |

| 业务模块边界 | 活动、表单、预约、订单之间存在交叉 | 建议统一用户、表单、订单、通知和核销底层模型 |
