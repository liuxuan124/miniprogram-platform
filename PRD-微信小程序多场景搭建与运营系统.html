1. 产品概述

本产品是一套面向品牌、商家、活动运营方的微信小程序搭建与运营管理系统。平台通过后台可视化配置能力，帮助运营人员完成小程序页面搭建、内容发布、会员运营、商品交易、订单处理、营销活动、活动报名、预约服务、素材管理和 AI Agent 配置。

产品核心价值是降低小程序运营和迭代成本，让非技术运营人员可以通过后台完成页面装修、业务配置和内容更新，并让商家在一个后台中完成从获客、转化、交易、履约到复购的闭环管理。

2. 产品目标

运营提效

支持运营人员完成小程序页面创建、装修、预览、保存草稿、版本管理和发布上线。

业务闭环

覆盖内容、表单、会员、用户、商品、订单、营销、活动、预约等核心运营场景。

端到端体验

支持用户在小程序端浏览内容、查看权益、购买商品、提交表单、预约服务、参与活动和咨询 AI。

智能化运营

支持 AI Agent 从模型接入、Prompt、知识库、沙盒测试、发布到运营监控的完整生命周期。

6. 核心业务流程

页面发布流程

  1. 进入页面管理，新建页面或选择已有页面。
  2. 进入页面装修器，添加 Banner、导航、商品、文章、活动入口等组件。
  3. 配置组件属性、数据源、样式和跳转。
  4. 保存草稿并预览小程序效果。
  5. 发布上线，小程序端展示新版本。

商品交易流程

  1. 商务运营创建实物、数字或服务商品。
  2. 商品上架后在小程序商城展示。
  3. 用户进入详情、确认订单并支付。
  4. 系统接收微信支付回调。
  5. 后台完成发货、核销、服务确认或退款审核。

活动报名流程

  1. 运营人员创建活动并配置时间、地点、名额和表单。
  2. 活动入口展示到首页或专题页。
  3. 用户查看活动详情并提交报名。
  4. 后台审核报名，通过后触发订阅消息。
  5. 现场或线上完成活动核销。

AI Agent 发布流程

  1. 选择模型并配置 API Key。
  2. 编写 Prompt、欢迎语和行为策略。
  3. 上传知识库并完成向量化。
  4. 沙盒测试典型问题并完成评估。
  5. 发布、灰度、回滚并监控运营数据。

10. 非功能需求

性能

后台首屏加载不超过 3 秒；列表查询 P95 不超过 800ms；页面发布后配置生效不超过 30 秒。

可用性

关键操作需有明确反馈；删除、发布、退款、切换生产支付环境等高风险操作必须二次确认。

安全

AppSecret、API Key、支付密钥和证书需加密存储、脱敏展示；支付回调需校验签名。

12. 验收标准

  1. 用户可以通过登录页进入后台，并完成主要导航切换。
  2. 工作台能够展示指标、趋势、待办和快捷入口。
  3. 页面管理可以新建页面、进入装修器、保存草稿、预览并发布。
  4. 页面装修器能够完成组件选择、结构展示、属性配置和手机预览。
  5. 商品交易流程至少覆盖商品详情、确认订单、支付成功、后台发货/核销/退款审核。
  6. 系统设置至少覆盖小程序、支付、物流、订阅消息、导航、插件和权限。
  7. AI Agent 至少覆盖模型接入、Prompt、知识库、沙盒测试、发布和监控。
  8. 高风险操作具备确认或明确提示，敏感信息脱敏展示。

14. 风险与待确认问题

风险/问题说明建议
页面装修器复杂度真实拖拽、排序、组件嵌套和实时预览复杂度较高。先做固定组件和基础排序，再迭代高级能力。
微信支付合规真实支付涉及商户资质、证书、回调、安全校验。尽早完成沙盒和生产资质确认。
AI Agent 合规AI 可能生成不准确承诺或泄露敏感信息。增加安全策略、敏感词、知识库引用和人工兜底。
小程序端动态渲染页面配置需转化为小程序可渲染结构。明确页面 DSL 和组件 schema。
业务模块边界活动、表单、预约、订单之间存在交叉。统一用户、表单、订单、通知和核销底层模型。
后续建议：页面搭建、营销、AI 推荐均按完整增强版目标设计，不再拆分多个产品层级；同时优先定义页面装修器组件 schema，并为商品、订单、表单、活动、预约、AI Agent 分别补充接口清单。
