## 同步前必须修改的小程序生产接口

当前小程序请求地址在：

```js
// miniapp/utils/request.js
const BASE_URL = 'http://127.0.0.1:8080'
```

上线前必须改成生产 HTTPS API 域名，例如：

```js
const BASE_URL = 'https://api.zfculture.site'
```

注意：

- 不能使用 `127.0.0.1`、`localhost` 作为线上小程序接口。
- 生产域名必须是 HTTPS。
- 生产域名必须配置到微信公众平台的 request 合法域名。

## 服务器环境变量

```env
WX_MINIAPP_APPID=wxea3928e0978492fe
WX_MINIAPP_SECRET=不要写入文档或日志
JWT_SECRET=生产强随机值
MYSQL_ROOT_PASSWORD=生产强密码
MYSQL_DATABASE=miniprogram
MYSQL_USER=miniprogram
MYSQL_PASSWORD=生产强密码
REDIS_PASSWORD=生产强密码
WX_PAY_NOTIFY_URL=https://api.zfculture.site/api/v1/pay/notify
```

AppSecret 只应写入服务器 `.env` 或后台系统配置，不要提交到 Git，也不要写入公开文档。

## 数据库配置同步

服务器数据库需要有以下配置：

- `wx_appid`
- `wx_app_secret`
- `tabbarItems`

推荐通过管理后台“系统设置”保存，避免 PowerShell 或 SQL 字符集造成中文乱码。

底部导航建议配置为：

```json
[
  {"text":"首页","pagePath":"pages/index/index"},
  {"text":"内容","pagePath":"pages/content-list/content-list"},
  {"text":"会员","pagePath":"pages/member-center/member-center"},
  {"text":"我的","pagePath":"pages/mine/mine"}
]
```

## 推荐部署方式：Docker Compose

服务器上执行：

```bash
cd /path/to/project/deploy
cp .env.example .env
vi .env
chmod +x scripts/deploy.sh scripts/rollback.sh
./scripts/deploy.sh
```

如果代码已经构建好，也可以用：

```bash
docker compose --env-file .env up -d --build
```

部署前建议先备份：

```bash
mysqldump -u root -p miniprogram > backup-miniprogram-$(date +%Y%m%d-%H%M%S).sql
docker compose ps
```

## 非 Docker 部署方式

后端：

```bash
cd backend
mvn -DskipTests package
java -jar target/*.jar
```

管理后台：

```bash
cd admin
npm ci
npm run build
```

然后把 `admin/dist` 部署到 Nginx 静态目录。

小程序：

1. 将 `miniapp/utils/request.js` 的 `BASE_URL` 改成生产 HTTPS API 域名。
2. 微信开发者工具打开 `miniapp` 目录。
3. 预览测试登录、首页、内容、会员、我的、商品详情。
4. 上传代码。
5. 提交审核。
6. 审核通过后发布。

## 风险点

1. 如果只同步后端和后台，不修改小程序 `BASE_URL`，线上小程序仍然无法访问服务器。
2. 如果服务器未配置正确 AppID/AppSecret，会再次出现 `invalid appid` 或登录失败。
3. 如果微信公众平台未配置 HTTPS 合法域名，真机会拦截接口请求。
4. 如果直接用命令行写入中文 JSON，可能出现底部导航中文乱码，建议通过后台 UI 保存。
5. 当前开发者工具使用基础库 3.16.0 灰度版，正式发布前建议切到稳定基础库再测一次。

## 建议的下一步

先确认服务器部署方式和生产 API 域名。确认后再做两件事：

1. 把小程序请求地址从本地地址切到生产 HTTPS API。
2. 按 Docker Compose 或现有服务器方式同步后端、后台和小程序代码。
